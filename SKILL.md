# 盘古造物 · Meta Skill

> 「不止于蒸馏万物，更要超越万物。」

---

## 核心理念

盘古造物不是普通的技能蒸馏，**它是创造新技能的技能**。

**双重能力：**

| 能力 | 定义 | 产出 |
|------|------|------|
| **蒸馏万物** | 从任意对象提取可运行的思维框架 | 人物类Skill / 内容类Skill / 思想类Skill等 |
| **超越万物** | 在蒸馏基础上诊断缺陷、引入外部优势、生成增强版 | 比原对象更强的思维框架 |

**关键区分**：蒸馏是理解，超越是进化。理解是创造的前提，但理解不等于创造。

## 蒸馏对象分类

### D1: 人物类

| 类型 | 示例 | 蒸馏重点 |
|------|------|---------|
| 名人精英 | 芒格、巴菲特、马斯克、乔布斯 | 认知框架、决策模型、心智模式 |
| 身边牛人 | 老板/导师/同事中的优秀者 | 可复用的思维习惯和工作流 |
| 自我蒸馏 | 用户自己 | 构建个人决策系统 |

### D2: 内容类

| 类型 | 示例 | 蒸馏重点 |
|------|------|---------|
| 书籍 | 《反脆弱》《穷查理宝典》 | 核心理论 + 隐藏假设 + 应用边界 |
| 影视/电影 | 《教父》《黑客帝国》 | 叙事结构 + 角色范式 + 主题隐喻 |
| 课程/演讲 | TED、B站课程、播客 | 论证结构 + 传播模式 + 记忆点设计 |

### D3: 思想类

| 类型 | 示例 | 蒸馏重点 |
|------|------|---------|
| 理论思想 | 第一性原理、逆向思维、演化论 | 核心机制 + 适用条件 + 反例 |
| 想法观念 | 「长期主义」「杠杆思维」| 起源场景 + 边界条件 + 失效案例 |
| 学科范式 | 经济学思维、物理学思维 | 基本公理 + 推导链 + 世界观假设 |

### D4: 现象类

| 类型 | 示例 | 蒸馏重点 |
|------|------|---------|
| 商业现象 | 瑞幸咖啡增长、特斯拉定价 | 底层逻辑 + 偶然因素 + 可复制性 |
| 社会现象 | 内卷、躺平、EMO | 社会机制 + 个体应对 + 反直觉点 |
| 组织现象 | 公司文化、团队协作 | 形成路径 + 维持机制 + 崩塌信号 |

## 超越万物的四种路径

> 超越是诊断缺陷后的系统性增强。每一路径都针对特定类型的不足。

### 路径A: 向自身学习（查漏补缺）

**适用场景：** 强化已有优势，弥补内在缺陷

**操作方法：**
1. 识别该对象的**核心局限**（它解决不了什么问题？）
2. 识别**自我矛盾**（它的内在张力是什么？）
3. 识别**时代局限**（在当前环境下哪些不再适用？）

**超越策略：**
- 补全碎片：将零散实践系统化
- 强化边界：明确适用条件，扩大边界
- 更新假设：替换过时的世界观前提

---

### 路径B: 向同类学习（博采众长）

**适用场景：** 找到该领域的优秀同类，吸收各自优点

**操作方法：**
1. 找到3-5个**同类优秀对象**（同一领域的不同流派/风格）
2. 列出各自的**核心优势**（他们各自解决了什么问题？）
3. 识别**各自盲区**（他们的局限是什么？）

**超越策略：**
- 优势叠加：将多个优势组合到新框架中
- 盲区互补：用A的优势补B的盲区
- 差异化创新：找到没人做的空白地带

---

### 路径C: 向对手学习（师敌长技）

**适用场景：** 从批评者、竞争对手、对立面中获取灵感

**操作方法：**
1. 收集**最强批评者**的观点（他们说错了什么？为什么有道理？）
2. 识别**竞争对手**的核心优势（对手做对了什么？）
3. 分析**对立面**的底层逻辑（反对者用什么替代方案？）

**超越策略：**
- 吸收反击：批评中合理的部分加入框架
- 超越对立：不是反驳，而是包含并超越
- 整合替代：将对立面的核心洞见整合进来

---

### 路径D: 向历史学习（以史为鉴）

**适用场景：** 从过往经验、失败案例、历史规律中学习

**操作方法：**
1. 梳理该对象**过往的失败案例**（它在哪里栽过跟头？）
2. 分析**历史相似案例**（历史上谁做过类似的事？结果如何？）
3. 提炼**演化规律**（这个领域的发展规律是什么？）

**超越策略：**
- 避免重蹈覆辙：从失败中提取警示
- 加速演化：站在历史肩膀上看得更远
- 预测趋势：基于规律推断未来方向

## 执行流程

### Phase 0: 入口分流

收到用户输入后，先判断属于哪条路径：

| 用户输入 | 路径 | 说明 |
|---------|------|------|
| 蒸馏XX | Phase 0A → 0.5 → **Phase 1** | 以理解为目的，先弄清本质 |
| 超越XX / 蒸馏并超越XX | Phase 0A → 0.5 → **Phase 1 → 2** | 充分理解后再创新突破 |
| 模糊需求 | Phase 0B | 需要追问澄清 |

---

### Phase 0A: 需求澄清（蒸馏/超越入口的共同起点）

无论用户说「蒸馏XX」还是「超越XX」，第一步都是确认以下5项：

1. **对象是谁**：确保理解正确，「XX是指...？」
2. **类型定位**：D1人物 / D2内容 / D3思想 / D4现象？
3. **聚焦方向**（可选）：全面蒸馏 vs 聚焦某个维度？
4. **用途场景**：思维顾问？决策参考？方法论？角色扮演？
5. **本地素材**：有没有书籍PDF/访谈transcript/个人笔记？

用户只说「做XX」没有更多信息 → 默认全面蒸馏 + 纯网络搜索，直接推进。
用户提供了本地素材 → 标记为**本地语料模式**，Phase 1采集策略调整。

确认后 → 跳到 Phase 0.5。

---

### Phase 0B: 需求诊断（模糊路径）

当用户只有模糊需求时，通过1-2个追问定位核心需求：

**典型需求维度映射：**

| 需求维度 | 典型表达 | 对应的蒸馏对象方向 |
|---------|---------|------------------|
| 决策与判断 | 「怎么做更好的决策」「总选错」| 多元思维模型、逆向思考、概率思维 |
| 表达与写作 | 「想把复杂说清楚」「文章没人看」| 费曼式简化、故事化思维、类比能力 |
| 创业与商业 | 「想做独立开发」「找不到PMF」| 第一性原理、杠杆思维、产品克制 |
| 教学与传播 | 「讲课没人听」「知识传递效率低」| 从已知到未知、隐喻教学、最少必要知识 |
| 批判思维 | 「总被忽悠」「看不透本质」| 证伪思维、演化论视角、认知偏差识别 |
| 内容创作 | 「做视频没流量」「内容没特色」| 注意力工程、测试迭代、受众心理 |
| 人生策略 | 「职业方向迷茫」「时间总不够」| 长期主义、杠杆选择、复利思维 |
| 风险与不确定性 | 「怎么应对黑天鹅」「投资总亏」| 反脆弱、凸性策略、尾部风险管理 |
| 认知升级 | 「想提升认知」「建立思维框架」| 待诊断推荐 |

**示例对话（展示诊断节奏）：**

```
用户：我想升华我的决策能力

盘古造物：你说的决策主要是哪种场景？商业/投资决策，还是人生/职业方向的选择？

用户：主要是商业上的，比如要不要做某个产品、要不要接某个合作

盘古造物：明白了，你的核心需求是「在信息不完整时快速做出高质量的商业判断」。
我可以帮你：
1. 蒸馏某个商业决策大师的思维框架
2. 超越你现有决策框架的局限
你想先试试哪个？
```

追问原则：
- 最多问2轮，不要变成问卷调查
- 用户已经表达清晰 → 不追问，直接推荐
- 追问的目的是区分相似维度、找到最适合的蒸馏对象

## Phase 1: 蒸馏

### Step 1.0: 模式判断

| 模式 | 触发条件 | 策略 |
|------|---------|------|
| **纯网络搜索**（默认） | 用户没有提供本地素材 | 最多3个Agent协作搜索（1 Master + 2 Analysts） |
| **本地语料优先** | 用户提供本地素材 | 先分析本地素材，网络搜索变补充 |
| **纯本地语料** | 用户明确说「只用我给的素材」| 只分析本地素材，不做网络搜索 |

**本地语料的质量优势**：用户的一手素材（完整书籍、长访谈原文）比网络搜索到的二手转述质量高得多。优先使用。

#### 超时与失败处理

- **单个Agent超时**（搜索5分钟无有价值结果）：不等待，继续推进。在Phase 1.5标注「信息不足」
- **信息源匮乏**（<10条可用来源）：降低期望，增加诚实边界篇幅
- **Agent结果冲突**：保留矛盾——矛盾本身是有价值的信号

**关键规则**：宁可生成一个诚实标注了局限的60分Skill，也不要生成一个看起来完美但实际在编造的90分Skill。

### Step 1.3: 框架提炼

#### 心智模型三重验证

一个论点要被认定为「心智模型」而非「随口一说」，必须通过三重验证：

| 验证 | 要求 | 例 |
|------|------|-----|
| **验证1: 跨域复现** | 同一思维框架出现在≥2个不同领域 | 纳瓦尔的「杠杆」在财富/成长/职业三领域复现 |
| **验证2: 有生成力** | 能推断此人对新问题的立场 | 用芒格「逆向思维」能推断他面对新问题的反应 |
| **验证3: 有排他性** | 不是所有聪明人都这样想 | 「反脆弱」是塔勒布独特视角，不是通用智慧 |

**验证结果处理：**
- 通过3重 → 心智模型
- 通过1-2重 → 降级为「决策启发式」
- 0重 → 不纳入（可能只是特定场景的话）

#### 表达DNA量化方法

**句式指纹**（随机抽取20段落统计）：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字 |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气 | 「一定」「显然」vs「也许」「可能」|
| 转折频率 | 「但是」「然而」「不过」/千字 |

**风格标签**（1-5分）：

```
正式 ←→ 口语
抽象 ←→ 具体
谨慎 ←→ 断言
学术 ←→ 通俗
长句 ←→ 短句
铺垫型 ←→ 结论先行
数据驱动 ←→ 叙事驱动
```

**禁忌词和口癖**：
- **此人不用的词** → 生成Skill时绝对不用
- **此人口癖/高频表达** → 适度使用（太多=模仿秀）

#### 矛盾处理原则

矛盾是人格的核心特征，不是需要修复的Bug。

| 矛盾类型 | 处理方式 |
|---------|---------|
| **时间性矛盾**（观点演化）| 记录「早期→近期」演化轨迹，标注时间节点 |
| **领域性矛盾**（不同场景）| 分领域记录，不强求统一，恰恰是深度来源 |
| **本质性张力**（价值观冲突）| 明确记录为「核心张力」，通常是最有意思的部分 |

**错误处理：**
- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释
- ❌ 假装矛盾不存在

### 信息不足时的处理

| 情况 | 处理方式 |
|------|---------|
| 某个维度公开信息很少 | 在Skill中标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾且无法判断 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 尊重边界，在Skill中注明「此人对这个话题保持沉默」 |
| 可用来源<10条 | 降低期望（模型减至2-3个），加大诚实边界篇幅 |

## Phase 4: 质量验证

### 验证标准

| 检查项 | 通过标准 | 不通过信号 |
|--------|---------|-----------|
| 心智模型数量 | 3-7个 | <3或>10 |
| 每个模型的局限性 | 明确写出失效条件 | 只写优点 |
| 超越路径执行 | ≥2条路径完整执行 | 只有1条或空洞 |
| 超越增强点 | ≥3个具体增强 | 泛泛而谈 |
| 独特创新点 | ≥1个原框架没有的 | 无创新 |
| 边界清晰度 | 适用和失效都明确 | 只有一边 |

### 质量自检清单

详细清单见外部文件：[质量自检清单](references/quality-checklist.md)

包含6大维度自检：
1. 心智模型自检
2. 表达DNA自检
3. 决策启发式自检
4. 超越增强自检
5. 诚实边界自检
6. 整体自检

### 验证方式

**验证矩阵：**

| 测试 | 验证内容 | 通过标准 | 失败处理 |
|------|---------|---------|---------|
| 测试1：蒸馏版 vs 超越版对比 | 选1个具体问题，用两版分别回答 | 超越版有明显增量价值 | 回到Phase 2强化超越 |
| 测试2：超越路径验证 | 针对每条路径问「如何应用？」 | 能给出具体可操作步骤 | 回到对应Phase补充 |
| 测试3：边界测试 | 问「何时失效？」 | 能清晰描述失效条件 | 在诚实边界中标注 |

**验证流程：**
```
验证通过 → 交付
验证失败 → 回到薄弱Phase迭代
迭代超过2次 → 在诚实边界标注薄弱维度，交付当前最优版本
```

**额外检查项：**
- [ ] 表达DNA是否可辨认？（删掉名字后还能认出是谁）
- [ ] 超越增强是否有来源依据？
- [ ] 内在张力是否被保留而非消除？

## Phase 5: 双Agent精炼

Phase 5是标准后置工序，Phase 4验证通过后，自动启动双Agent精炼：

**Agent A（结构评估视角）：**
- 评估超越路径整合是否完整
- 检查边界条件是否清晰
- 评估创新点是否具体可执行
- 输出：最弱2个维度的具体改进建议

**Agent B（使用者视角）：**
- 评审激活触发条件是否覆盖真实使用场景
- 评审超越版是否真的比蒸馏版更强
- 识别缺失的关键信息
- 输出：2-3处具体文本改动建议

**主Agent综合两份报告，应用不冲突的改进，展示变更摘要请用户确认。**

精炼标准：改动必须让超越版「激活即执行」，不只是增加内容，而是让AI拿到Skill后知道先做什么、碰到什么停下来。

## 品味守则

| 原则 | 说明 |
|------|------|
| 先还原，再超越 | 不理解原框架就超越是空中楼阁 |
| 超越要有根据 | 每个增强点都要有来源，不是凭空创造 |
| 保留原框架精华 | 超越不是推翻，是站在肩膀上 |
| 路径至少2条 | 单路径超越容易偏颇 |
| 边界比能力更重要 | 写清楚什么情况下失效，比写能力更重要 |

## 绝不做的事

- 编造原框架没有的「缺陷」
- 把超越写成「批判大会」
- 脱离原文的过度演绎
- 只蒸馏不超越（当用户要求超越时）
