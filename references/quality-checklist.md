## 1. 心智模型自检

| 检查项 | 说明 |
|--------|------|
| [ ] 每个模型都有至少2个不同领域的证据 | 跨域复现验证 |
| [ ] 模型数量在3-7个之间 | 太少=太浅，太多=没提炼 |
| [ ] 每个模型都有明确的应用场景和局限 | 不能放之四海皆准 |
| [ ] 模型之间有张力但不矛盾 | 内在张力是真实的表现 |
| [ ] 通过了三重心智模型验证 | 跨域复现+生成力+排他性 |

## 2. 表达DNA自检

| 检查项 | 说明 |
|--------|------|
| [ ] 读起来有辨识度，不像通用AI | 一眼可辨认 |
| [ ] 没有过度模仿变成caricature | 抓核心而非表面 |
| [ ] 抓住了核心特征而非表面模仿 | 是灵魂不是形式 |
| [ ] 禁忌词已标注 | 避免使用的表达 |

---

## 3. 决策启发式自检

| 检查项 | 说明 |
|--------|------|
| [ ] 每条规则都有具体案例支撑 | 不能是空泛原则 |
| [ ] 可以被新情况触发 | 不只适用于原始案例 |

---

## 4. 超越增强自检

| 检查项 | 说明 |
|--------|------|
| [ ] 每个增强点都标注了来源路径 | A/B/C/D路径溯源 |
| [ ] 增强有具体案例而非泛泛而谈 | 有血有肉 |
| [ ] 独特创新点≥1个 | 不是简单叠加 |

---

## 5. 诚实边界自检

| 检查项 | 说明 |
|--------|------|
| [ ] 明确写了做不到什么 | 边界比能力更重要 |
| [ ] 标注了调研时间 | 信息时效性 |
| [ ] 承认了信息不足的维度 | 不装作完美 |

---

## 6. 整体自检

| 检查项 | 说明 |
|--------|------|
| [ ] 用此人的眼睛看一个新问题，能得到有价值的视角？ | 框架运行测试 |
| [ ] 不是此人原话的拼凑，而是框架的运行？ | 不是复读机 |
| [ ] 删掉名字后，还能认出这是谁的思维方式？ | 核心辨识度 |

---

## 快速检查脚本

完成Skill后，用以下问题快速验证：

```
1. 把Skill给一个了解此人的人看，他能认出是谁吗？
2. 把Skill给一个不了解此人的人看，他能得到有价值的视角吗？
3. 问一个Skill中没有明确回答的新问题，框架能推导出一致的答案吗？
```

如果任何一个答案是否定的，说明还有改进空间。
